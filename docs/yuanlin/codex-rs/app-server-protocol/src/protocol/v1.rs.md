# `v1.rs` — v1 遗留 API 协议类型

## 功能概述

定义了 app-server 协议 v1 版本的所有类型，包括初始化参数/响应、客户端信息、会话摘要、认证状态、命令执行审批、补丁审批、Git diff、用户保存配置等。这些是最初的 app-server 协议类型，v2 版本对其中大部分进行了重新设计，但仍需保持向后兼容。

## 关键结构体/枚举

| 名称 | 类型 | 说明 |
|------|------|------|
| `InitializeParams` | 结构体 | 初始化请求参数（客户端信息 + 能力协商） |
| `InitializeResponse` | 结构体 | 初始化响应（用户代理、Codex 主目录、平台信息） |
| `InitializeCapabilities` | 结构体 | 客户端能力（实验性 API 支持、通知方法退出列表） |
| `ConversationSummary` | 结构体 | 会话摘要（ID、路径、预览、模型、CWD、Git 信息等） |
| `ExecCommandApprovalParams` | 结构体 | 命令执行审批参数 |
| `ApplyPatchApprovalParams` | 结构体 | 补丁应用审批参数 |
| `UserSavedConfig` | 结构体 | 用户保存的配置（审批策略、沙箱模式、模型、工具等） |
| `SandboxSettings` | 结构体 | 沙箱设置（可写根、网络访问等） |

## 依赖关系

- **引用的 crate**: `codex_protocol`（多个子模块）、`codex_git_utils`、`codex_utils_absolute_path`、`schemars`、`serde`、`ts_rs`
- **被引用方**: `common.rs` 中通过 `client_request_definitions!` 和 `server_request_definitions!` 宏引用

## 实现备注

- `GetConversationSummaryParams` 使用 `#[serde(untagged)]` 支持按 rollout 路径或 ThreadId 两种方式查询。
- `ExecOneOffCommandParams` 是旧版命令执行参数，通过 `mappers.rs` 转换为 v2 的 `CommandExecParams`。
- `UserSavedConfig` 聚合了所有用户可配置项及命名配置文件（profiles）。
