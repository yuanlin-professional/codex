# `v2.rs` — v2 新版 API 协议类型

## 功能概述

定义了 app-server 协议 v2 版本的所有类型，是整个协议中最大的单文件（超过 8000 行）。涵盖了线程生命周期管理（创建/恢复/分支/归档/卸载/回滚）、Turn 生命周期（启动/引导/中断/完成）、项目类型（命令执行、文件变更、MCP 工具调用、动态工具调用、网页搜索、图片生成、推理等）、账户管理（登录/登出/认证刷新）、配置读写、文件系统操作、模型列表、实验性功能、技能/插件管理、实时音频/文本、审阅等全部 v2 API 的请求参数和响应类型。

## 关键结构体/枚举

| 名称 | 类型 | 说明 |
|------|------|------|
| `Thread` | 结构体 | 线程完整状态（ID、预览、状态、CWD、Turn 列表等） |
| `Turn` | 结构体 | 单次 Turn（ID、状态、项目列表、错误等） |
| `ThreadItem` | 枚举 | 线程中的所有项目类型联合（CommandExecution / PatchApply / McpToolCall / Message 等） |
| `ThreadStatus` | 枚举 | 线程状态（Idle / Active / Stale / Error） |
| `TurnStartParams` | 结构体 | Turn 启动参数（线程 ID、输入、协作模式等） |
| `Account` | 枚举 | 账户类型（ApiKey / Chatgpt / ChatgptAuthTokens） |
| `SandboxPolicy` | 枚举 | v2 版沙箱策略 |
| `CommandAction` | 枚举 | 命令审批动作选项 |
| `ReviewDecision` | 枚举 | v2 版审批决策 |

## 关键函数/方法

| 函数 | 签名摘要 | 说明 |
|------|----------|------|
| 大量 `v2_enum_from_core!` 宏调用 | - | 从 `codex_protocol` 核心类型到 v2 API 类型的自动映射 |
| 各种 `From<Core*> for V2*` | - | 核心类型到 v2 wire 类型的转换实现 |

## 依赖关系

- **引用的 crate**: `codex_protocol`（几乎全部子模块）、`codex_experimental_api_macros`、`schemars`、`serde`、`ts_rs`、`thiserror`
- **被引用方**: `common.rs` 中宏生成的请求/通知定义、`thread_history.rs`

## 实现备注

- 使用 `v2_enum_from_core!` 宏批量生成 core 枚举到 v2 枚举的映射，统一 camelCase 命名。
- 所有 v2 类型的序列化均使用 camelCase，与 TypeScript 客户端约定一致。
- 实验性字段使用 `#[experimental(...)]` 属性标记，与 `ExperimentalApi` trait 集成。
- 文件体量极大，建议按功能区域（线程管理、Turn 管理、项目类型、账户管理等）分别理解。
