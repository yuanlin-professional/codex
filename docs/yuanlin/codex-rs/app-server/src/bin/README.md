# bin

## 功能概述

`bin/` 目录包含 `app-server` crate 中的辅助二进制工具程序。这些工具并非服务器本身，而是为集成测试和通知捕获等场景提供的独立小型可执行文件。

这两个工具都实现了相同的核心功能：将命令行传入的负载内容以原子方式写入指定的输出文件（先写临时文件再重命名），确保文件内容的一致性。

## 目录结构

```
bin/
├── notify_capture.rs           # 通知捕获工具
└── test_notify_capture.rs      # 测试用通知捕获工具
```

## 核心接口/API

### notify_capture

- **用途**：将通知负载原子性地写入指定文件
- **命令行参数**：`<输出文件路径> <负载内容>`
- **工作流程**：
  1. 接收输出路径和负载两个参数
  2. 创建临时文件（`.tmp` 后缀）
  3. 写入负载内容并调用 `sync_all()` 确保数据刷盘
  4. 通过 `fs::rename()` 原子性地将临时文件移动到目标路径
- **错误处理**：使用 `anyhow` 提供详细的上下文错误信息

### test_notify_capture

- **用途**：简化版的通知捕获工具，用于测试场景
- **命令行参数**：`<输出文件路径> <负载内容>`
- **工作流程**：
  1. 接收输出路径和负载两个参数
  2. 创建 `.json.tmp` 后缀的临时文件
  3. 写入负载内容
  4. 通过 `fs::rename()` 原子性地重命名为目标文件
- **与 notify_capture 的区别**：使用更简单的 `std::fs::write` 而非手动 `File::create` + `sync_all`，临时文件后缀为 `.json.tmp` 而非 `.tmp`
